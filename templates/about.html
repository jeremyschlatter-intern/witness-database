About This Database

What Is This?

The Congressional Witness Database is a searchable index of witnesses who have testified before House and Senate committees from the 116th Congress (2019) to the present. It lets you look up who testified, when, before which committees, and in what capacity.

This tool is designed for congressional staff, journalists, researchers, and anyone who needs to quickly answer questions like:

  • Who has served as FAA Administrator and when did they testify?
  • How many times has a particular person testified before Congress?
  • Which witnesses appeared before the House Judiciary Committee this year?
  • What hearings covered a specific topic?
  • Which organizations send the most witnesses to Congress?

Data Sources

All data comes from official government APIs:

  • Congress.gov Committee Meeting API — Structured witness data including names, titles, organizations, and links to written statements, biographies, and truth-in-testimony disclosures. This provides comprehensive witness data for House hearings.
  • Congress.gov Hearing API — Hearing metadata and links to GovInfo transcripts.
  • GovInfo API — Full hearing transcripts in HTML and PDF formats.

Data is refreshed periodically. Each witness has a stable unique ID (e.g., W-00001) for reliable cross-referencing.

Features

  • Smart search: Search by witness name, hearing topic, title/position, organization, or agency abbreviation (e.g., FAA, FBI, EPA, SEC). The search engine automatically expands abbreviations to their full names.
  • Title tracking: See who has held a given title (e.g., "Administrator") and when they testified in that capacity. Browse all tracked titles and positions.
  • Document links: Direct links to written statements (PDFs), witness biographies, and truth-in-testimony disclosures hosted on Congress.gov.
  • Committee pages: View each committee's hearing history, top witnesses, and filtering options.
  • Statistics dashboard: Explore top organizations, most active committees, and repeat witnesses.
  • CSV export: Download witness data as CSV for your own analysis.
  • Direct links: Every hearing links to the original Congress.gov event page.
  • JSON API: Programmatic access to all data for integration with other tools.

Known Limitations

  • Senate witness data: The Congress.gov Committee Meeting API does not provide structured witness information for most Senate hearings. Senate hearings are listed, but witnesses may not be indexed unless the API provides them. A future update will parse witness names from hearing transcripts to fill this gap.
  • 117th Congress House data: The Congress.gov API currently returns an error for 117th Congress House committee meetings, so this data is unavailable.
  • Witness deduplication: The same person may appear under slightly different name variants. We use conservative matching to merge obvious duplicates (e.g., "Steve" and "Stephen" Dickson), but some edge cases may remain.
  • Testimony text: Written statement PDFs are linked but their text content is not yet extracted into the database for full-text search. This is planned for a future update.
  • Questions for the Record (QFRs): QFR extraction from hearing transcripts is not yet implemented. The database schema supports QFRs and they will be added in a future update.

API Access

This database provides a JSON API for programmatic access. All endpoints return JSON.

Endpoint Description Parameters
/api/stats Database statistics overview —
/api/witnesses List witnesses q (search), limit (max 500)
/api/witness/{id} Full witness detail with titles and appearances —
/api/hearing/{id} Full hearing detail with witnesses and committees —
/api/search Search witnesses and hearings q (required), limit (max 100)

Example: /api/search?q=FAA+Administrator returns witnesses and hearings matching "Federal Aviation Administration Administrator".

Open Source

This project was built as a demonstration of AI-assisted legislative technology. The data is sourced entirely from publicly available government APIs and is freely available through the API endpoints above.
